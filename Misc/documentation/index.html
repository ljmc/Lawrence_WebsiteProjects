Bell

Responsive Countdown Landing Page

  • Created: 11/29/2012
  • By: Sonny T.
  • Email: hi@sonnyt.com

Thank you for purchasing Bell. In this documentation you will find details on how to customize the template for your needs. It's fairly simple actually, template contains two plugins, Countdown and Twitter Feed, which both require jQuery.

Assets

Here are the description of the files and folders you find inside of an /assets directory;

/demo - This folder contains all the images used for the demo purposes, after you fully modify the content of the template, feel free to remove this directory.

/javascript/main.js - The main Javascript file that triggers Twitter, Countdown and Subscription functions.

/stylesheet/style.css - The main CSS file.

  • colors/ - Colors stylesheets
  • lib/ - Third-party CSS libraries
  • partials/ - Partials of the theme
  • themes/ - Themes of the theme

/video - This folder contains header video background.

Header

The Bell allows you to have video or image as your header background.

To use video header (video.html) background you need ".mp4" and ".ogg" files. And replace the "main.mp4" and "main.ogg" files inside of "assets/videos" directory.

To use image header (image.html) replace the "main.jpg" image inside of "assets/images" directory.

Colors

The Bell also has different accent color pallets that you can choose from. To change the accent color of your page, change the class "green-color" in "body" tag with either of these:

  • green-color
  • orange-color
  • blue-color
  • yellow-color

The theme also comes with multiple color versions, dark and blue.

To use the dark color theme, add "dark-theme" to the "body".
To use the blue color theme, add "blue-theme" to the "body".

Countdown Plugin

The countdown for demo sake comes with futureDate function which adds +60 days to current date, so you can remove the function.

To change the countdown clock open up a /assets/javascript/main.js file and go down to line 8:

date: '12/24/2016 12:00:00', // CHANGE THE DATE

Your date has to be in a format below:

MM/DD/YYYY HH:MM:SS

Here is an example:

date: '12/24/2012 12:00:00',

Twitter Plugin

Important: Due to recent changes in the Twitter API, the Twitter feed functionality now requires PHP. The files must be loaded from a server with PHP enabled.

Bell includes a script to display tweets. It can pull the latest tweets from your screen name.

To set it up you must optain a few keys from Twitter. Follow these steps to get them:

  • Go to https://dev.twitter.com/ and sign in with your twitter info.

  • In the upper right hand corner dropdown, select "My Applications"

  • Click the "Create a new Application" button.

  • Fill in the info for you website. Note: In the Callback URL field, fill in 

    http://yourwebsite.com/api/tweet.php
    Replacing "yourwebsite.com" with your website url.

  • In the following screen you will see a section for "OAuth". From here you will find your applications Consumer Key and Consumer Secret. Add these two in the corresponting constant in /api/config.php.

    Then hit the "Create Access Token" button.

  • Your access token should now be in the "Your Access Token" section. Add these in to the config file as well.

You should now have a functioning Twitter module.

Subscription Box

Subscription box is integrated with MailChimp. To get the form running, you just need your MailChimp API Key and List ID.

To get your API Key, click here.
To get your List ID, click here.

After you get your List ID and API Key, go to /api/config.php and change the lines 10 & 11.

	    define('MC_API_KEY', 'Replace this with the Mailchimp API key');
	    define('MC_LIST_ID', 'Replace this with the Mailchimp List Id');

Contact Us

Contact us form is very simple to setup. Just go to /api/config.php and change the line 14.

	    define('EMAIL', 'Replace this with your contact email');

Style Sheet

CSS style sheet is located inside of "assets" directory. Style Sheet is broken down into multiple small files to make it easier for you to focus and find parts of the theme.

style.css - main file that imports different partials, and it self is loaded by HTML.

/colors - theme colors are located in here

/lib - inside of "lib" directory you will find 3rd party CSS files, in this case, it's Twitter Bootstrap

/partials - different portions of the theme.
